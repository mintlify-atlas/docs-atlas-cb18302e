## Quick Start Guide

This guide will walk you through creating your first Discord bot with Seyfert in just a few minutes.

<Note>
  Before you begin, make sure you have Node.js v18 or higher installed. We recommend using the LTS version.
</Note>

<Steps>
  <Step title="Install Seyfert">
    Choose your favorite package manager to install Seyfert:
    
    <CodeGroup>
    ```bash npm
    npm install seyfert
    ```
    
    ```bash pnpm
    pnpm add seyfert
    ```
    
    ```bash yarn
    yarn add seyfert
    ```
    
    ```bash bun
    bun add seyfert
    ```
    
    ```bash deno
    deno add npm:seyfert
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Get Your Discord Bot Token">
    1. Go to the [Discord Developer Portal](https://discord.com/developers/applications)
    2. Click "New Application" and give it a name
    3. Go to the "Bot" section in the sidebar
    4. Click "Add Bot"
    5. Under the token section, click "Reset Token" and copy your bot token
    
    <Warning>
      Never share your bot token! Keep it secret and never commit it to version control.
    </Warning>
  </Step>
  
  <Step title="Create Your Bot Client">
    Create a new file called `index.ts` (or `index.js` if not using TypeScript):
    
    ```typescript index.ts
    import { Client, config } from 'seyfert';
    
    const client = new Client();
    
    client.start({
      connection: config.bot({
        token: process.env.DISCORD_TOKEN ?? '',
        intents: ['Guilds', 'GuildMessages', 'MessageContent']
      })
    });
    
    console.log('Bot is starting...');
    ```
    
    Create a `.env` file to store your token:
    
    ```bash .env
    DISCORD_TOKEN=your_bot_token_here
    ```
    
    <Tip>
      Install `dotenv` to load environment variables: `npm install dotenv`
    </Tip>
  </Step>
  
  <Step title="Create Your First Command">
    Create a `commands` directory and add a ping command:
    
    ```typescript commands/ping.ts
    import { Command, type CommandContext } from 'seyfert';
    
    export default class PingCommand extends Command {
      name = 'ping';
      description = 'Check the bot latency';
      
      async run(ctx: CommandContext) {
        const latency = ctx.client.gateway.latency;
        await ctx.write({
          content: `🏓 Pong! Latency: ${latency}ms`
        });
      }
    }
    ```
    
    <Info>
      Commands are automatically discovered and loaded from your commands directory.
    </Info>
  </Step>
  
  <Step title="Add Event Handlers">
    Create an `events` directory and add a ready event:
    
    ```typescript events/ready.ts
    import { createEvent } from 'seyfert';
    
    export default createEvent({
      data: { name: 'botReady', once: true },
      run: (user, client) => {
        console.log(`✅ ${user.username} is online!`);
        console.log(`📊 Serving ${client.cache.guilds?.size ?? 0} guilds`);
      }
    });
    ```
  </Step>
  
  <Step title="Run Your Bot">
    If using TypeScript, compile and run:
    
    ```bash
    npx tsx index.ts
    ```
    
    Or if using JavaScript:
    
    ```bash
    node index.js
    ```
    
    Your bot should now be online! Invite it to your server and try the `/ping` command.
    
    <Tip>
      Use a process manager like PM2 for production deployments to keep your bot running reliably.
    </Tip>
  </Step>
</Steps>

## Example: A More Complete Bot

Here's a slightly more advanced example with multiple commands and error handling:

```typescript index.ts
import { Client, config } from 'seyfert';

const client = new Client();

// Load commands and events from directories
client.start({
  connection: config.bot({
    token: process.env.DISCORD_TOKEN ?? '',
    intents: [
      'Guilds',
      'GuildMessages',
      'MessageContent',
      'GuildMembers'
    ]
  }),
  // Specify directories for auto-loading
  commandsDir: './commands',
  eventsDir: './events',
  langsDir: './locales' // Optional: for i18n
}).then(() => {
  console.log('Bot started successfully!');
}).catch(console.error);
```

```typescript commands/greet.ts
import { Declare, Command, type CommandContext, Options, createStringOption } from 'seyfert';

const options = {
  name: createStringOption({
    description: 'Name to greet',
    required: true
  })
};

@Declare({
  name: 'greet',
  description: 'Greet a user by name'
})
@Options(options)
export default class GreetCommand extends Command {
  async run(ctx: CommandContext<typeof options>) {
    const name = ctx.options.name;
    await ctx.write({
      content: `👋 Hello, ${name}! Welcome to our server!`
    });
  }
}
```

## Common Issues

<AccordionGroup>
  <Accordion title="Bot doesn't respond to commands">
    Make sure you've:
    - Enabled the "Message Content" intent in the Developer Portal
    - Invited the bot with the `applications.commands` scope
    - Waited a few minutes for commands to register globally
    - Used the correct token in your `.env` file
  </Accordion>
  
  <Accordion title="Module not found errors">
    If you see module errors:
    - Make sure you installed all dependencies: `npm install`
    - If using TypeScript, ensure your `tsconfig.json` is properly configured
    - Check that file paths in your imports are correct
  </Accordion>
  
  <Accordion title="Permission errors">
    If the bot can't perform actions:
    - Check that the bot has the required permissions in your Discord server
    - Ensure you invited the bot with the correct OAuth2 scopes
    - Verify the bot's role is high enough in the role hierarchy
  </Accordion>
</AccordionGroup>
