## What is Seyfert?

Seyfert is a powerful, modern Discord bot framework built from scratch in TypeScript. Designed for developers who demand performance, type safety, and an exceptional developer experience, Seyfert provides everything you need to build production-ready Discord bots with confidence.

<Note>
  Seyfert is actively maintained and supports all the latest Discord features as soon as they're available.
</Note>

## Why Choose Seyfert?

Seyfert stands out from other Discord frameworks with its unique combination of features and philosophy:

### Low Resource Usage

Seyfert is engineered for efficiency with intelligent caching and memory management:

- **Smart Cache Control**: Granular control over what gets cached and how long it's stored
- **Memory Efficient**: Uses significantly less RAM compared to traditional frameworks
- **Optimized Data Structures**: Custom collection implementations for better performance

```typescript
import { Client } from 'seyfert';

const client = new Client({
  // Disable caching for resources you don't need
  cache: {
    disabledCache: {
      messages: true,  // Don't cache messages
      presences: true, // Don't cache presence updates
    }
  }
});
```

### Type-Safe by Design

Full TypeScript support with intelligent type inference throughout:

- **Command Options**: Automatically typed based on your option definitions
- **Context Extensions**: Type-safe custom context properties
- **Event Handlers**: Strongly typed event parameters
- **Builder Pattern**: Type-safe message and component builders

```typescript
import { Declare, Options, type CommandContext } from 'seyfert';
import { ApplicationCommandOptionType } from 'seyfert/types';

@Declare({
  name: 'greet',
  description: 'Greet a user'
})
@Options({
  user: {
    type: ApplicationCommandOptionType.User,
    description: 'User to greet',
    required: true
  }
})
export class GreetCommand {
  async run(ctx: CommandContext<typeof this.options>) {
    // ctx.options.user is automatically typed!
    const user = ctx.options.user;
    await ctx.write({ content: `Hello, ${user.username}!` });
  }
}
```

### Built-in Sharding

Scale your bot effortlessly with native sharding support:

- **Automatic Shard Management**: Handles shard lifecycle automatically
- **Worker-based Sharding**: Distribute shards across worker threads
- **Resharding Support**: Automatically adjust shard count as your bot grows
- **Shard-aware Caching**: Cache data efficiently across shards

### Modern Developer Experience

Seyfert provides tools and patterns that make development enjoyable:

- **Decorator-based Commands**: Clean, declarative command definitions
- **Middleware System**: Reusable logic for authentication, rate limiting, and more
- **Event System**: Simple, type-safe event handling
- **File-based Loading**: Automatic discovery of commands and events
- **Hot Reload**: Reload commands without restarting your bot (in development)

### Latest Discord Features

Seyfert stays up-to-date with Discord's API:

- ✅ Application Commands (Slash Commands, Context Menus)
- ✅ Message Components (Buttons, Select Menus, Modals)
- ✅ Threads and Forum Channels
- ✅ Voice States and Stage Instances
- ✅ AutoMod Integration
- ✅ Application Entry Points
- ✅ Polls and Media Galleries

## Architecture Overview

Seyfert follows a modular architecture that separates concerns and promotes maintainability:

```
┌─────────────────────────────────────────┐
│            Your Bot Code                │
│  (Commands, Events, Components)         │
└─────────────────┬───────────────────────┘
                  │
┌─────────────────▼───────────────────────┐
│         Seyfert Client                  │
│  • Command Handler                      │
│  • Event Handler                        │
│  • Component Handler                    │
└─────────────────┬───────────────────────┘
                  │
┌─────────────────▼───────────────────────┐
│         Gateway/Shard Manager           │
│  • WebSocket Connections                │
│  • Shard Distribution                   │
└─────────────────┬───────────────────────┘
                  │
┌─────────────────▼───────────────────────┐
│            Cache Layer                  │
│  • Memory Adapter (default)             │
│  • Redis Adapter (optional)             │
│  • Custom Adapters                      │
└─────────────────┬───────────────────────┘
                  │
┌─────────────────▼───────────────────────┐
│          REST API Handler               │
│  • Rate Limiting                        │
│  • Request Queue                        │
└─────────────────────────────────────────┘
```

## Runtime Support

Seyfert works across multiple JavaScript runtimes:

<Tabs>
  <Tab title="Node.js">
    **Recommended:** Node.js v18 or higher (LTS recommended)
    
    ```bash
    node --version  # Should be v18.0.0 or higher
    npm install seyfert
    ```
  </Tab>
  
  <Tab title="Bun">
    **Fully Supported:** Bun provides excellent performance
    
    ```bash
    bun --version
    bun add seyfert
    ```
  </Tab>
  
  <Tab title="Deno">
    **Supported:** Import via npm specifier
    
    ```bash
    deno add npm:seyfert
    ```
  </Tab>
</Tabs>

<Warning>
  Node.js v16 requires the `--experimental-fetch` flag, but v18+ is strongly recommended.
</Warning>

## Philosophy

Seyfert is built on these core principles:

1. **Developer Experience First**: Writing bot code should be enjoyable and intuitive
2. **Performance Matters**: Efficient bots can scale further and cost less
3. **Type Safety**: Catch errors at compile time, not runtime
4. **Modern JavaScript**: Leverage the latest language features and patterns
5. **Flexibility**: Provide sane defaults but allow customization when needed
