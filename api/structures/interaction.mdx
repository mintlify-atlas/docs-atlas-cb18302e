Seyfert provides various interaction classes for handling different types of Discord interactions. All interaction classes extend from `BaseInteraction`.

## BaseInteraction

The base class for all interactions.

```typescript
class BaseInteraction<FromGuild extends boolean = boolean>
```

### Properties

- `id: string` - Interaction ID
- `applicationId: string` - Application ID
- `type: InteractionType` - Type of interaction
- `token: string` - Interaction token
- `version: number` - Interaction version
- `guildId?: string` - Guild ID (if invoked in a guild)
- `channelId?: string` - Channel ID
- `locale?: string` - User's locale
- `guildLocale?: string` - Guild's locale
- `user: User` - User who triggered the interaction
- `member?: InteractionGuildMember` - Guild member (if in a guild)
- `channel?: AllChannels` - Channel where interaction was triggered
- `message?: Message` - Message associated with interaction (for components)
- `appPermissions: PermissionsBitField` - App permissions in the channel
- `entitlements: Entitlement[]` - Entitlements for the user
- `replied?: Promise<boolean> | boolean` - Whether interaction has been replied to
- `deferred?: boolean` - Whether interaction has been deferred

### Methods

#### reply

```typescript
reply<WR extends boolean = false>(
  body: ReplyInteractionBody,
  withResponse?: WR
): Promise<When<WR, WebhookMessage, undefined>>
```

Replies to the interaction.

**Parameters:**
- `body` - Reply data
- `withResponse` - Whether to return the message object

**Returns:** Promise resolving to message if `withResponse` is true

**Example:**
```typescript
await interaction.reply({
  type: InteractionResponseType.ChannelMessageWithSource,
  data: {
    content: 'Hello!'
  }
});
```

#### deferReply

```typescript
deferReply<WR extends boolean = false>(
  flags?: MessageFlags,
  withResponse?: WR
): Promise<When<WR, WebhookMessage, undefined>>
```

Defers the interaction reply (shows "Bot is thinking...").

**Parameters:**
- `flags` - Message flags (e.g., ephemeral)
- `withResponse` - Whether to return the message object

**Example:**
```typescript
await interaction.deferReply(MessageFlags.Ephemeral);
// Do some async work...
await interaction.editResponse({ content: 'Done!' });
```

#### fetchGuild

```typescript
fetchGuild(mode?: 'rest' | 'flow'): Promise<Guild | undefined>
fetchGuild(mode: 'cache'): Guild | undefined
```

Fetches the guild where the interaction was triggered.

**Example:**
```typescript
const guild = await interaction.fetchGuild();
if (guild) {
  console.log(guild.name);
}
```

### Type Guards

```typescript

interaction.isMessage(): interaction is MessageCommandInteraction

interaction.isModal(): interaction is ModalSubmitInteraction
interaction.isEntryPoint(): interaction is EntryPointInteraction
```

## Interaction

Extends `BaseInteraction` with response management methods.

```typescript
class Interaction extends BaseInteraction
```

### Methods

#### write

```typescript
write<FR extends boolean = false>(
  body: InteractionCreateBodyRequest,
  withResponse?: FR
): Promise<When<FR, WebhookMessage, void>>
```

Sends a response to the interaction.

**Example:**
```typescript
await interaction.write({
  content: 'Response!',
  embeds: [{ title: 'Embed' }]
});
```

#### modal

```typescript
modal(body: ModalCreateBodyRequest, options?: undefined): Promise<undefined>
modal(body: ModalCreateBodyRequest, options: ModalCreateOptions): Promise<ModalSubmitInteraction | null>
```

Shows a modal to the user. Can optionally wait for submission.

**Example:**
```typescript
// Show modal without waiting
await interaction.modal({
  title: 'Feedback Form',
  custom_id: 'feedback',
  components: [/* ... */]
});

// Show modal and wait for submission
const modalSubmit = await interaction.modal(
  { /* modal data */ },
  { waitFor: 60000 } // Wait up to 60 seconds
);

if (modalSubmit) {
  const feedback = modalSubmit.getInputValue('feedback_input', true);
  console.log('User feedback:', feedback);
}
```

#### editOrReply

```typescript
editOrReply<FR extends boolean = false>(
  body: InteractionCreateBodyRequest,
  fetchReply?: FR
): Promise<When<FR, WebhookMessage, void>>
```

Edits the response if already replied, otherwise sends a new reply.

**Example:**
```typescript
await interaction.editOrReply({
  content: 'Updated or new message'
});
```

#### editResponse

```typescript
editResponse(body: InteractionMessageUpdateBodyRequest): Promise<WebhookMessage>
```

Edits the original interaction response.

**Example:**
```typescript
await interaction.deferReply();
// Do work...
await interaction.editResponse({
  content: 'Work complete!'
});
```

#### fetchResponse

```typescript
fetchResponse(): Promise<WebhookMessage>
```

Fetches the original interaction response.

#### deleteResponse

```typescript
deleteResponse(): Promise<void>
```

Deletes the original interaction response.

**Example:**
```typescript
await interaction.deleteResponse();
```

#### followup

```typescript
followup(body: MessageWebhookCreateBodyRequest): Promise<WebhookMessage>
```

Sends a followup message.

**Example:**
```typescript
await interaction.reply(/* initial response */);
await interaction.followup({
  content: 'This is a followup message'
});
```

## ComponentInteraction

Base class for all component interactions.

```typescript
class ComponentInteraction extends Interaction
```

**Type:** `InteractionType.MessageComponent`

### Properties

- `data: { customId: string; componentType: ComponentType; ... }` - Component data
- `message: Message` - Message containing the component
- `customId: string` - The component's custom ID
- `componentType: ComponentType` - Type of component

### Methods

#### update

```typescript
update(data: ComponentInteractionMessageUpdate): Promise<void>
```

Updates the message containing the component.

**Example:**
```typescript
await interaction.update({
  content: 'Button clicked!',
  components: [] // Remove components
});
```

#### deferUpdate

```typescript
deferUpdate(): Promise<void>
```

Defers the component update (acknowledges without changing the message).

**Example:**
```typescript
await interaction.deferUpdate();
// Do async work...
await interaction.editResponse({ content: 'Done!' });
```

## AutocompleteInteraction

Represents an autocomplete interaction.

```typescript
class AutocompleteInteraction extends BaseInteraction
```

**Type:** `InteractionType.ApplicationCommandAutocomplete`

### Properties

- `data: { id: string; name: string; options: CommandOption[]; ... }` - Command data
- `options: OptionResolver` - Option resolver for accessing option values

### Methods

#### getInput

```typescript
getInput(): string
```

Gets the current autocomplete input value.

**Returns:** The focused option's current value

#### respond

```typescript
respond(choices: { name: string; value: string | number }[]): Promise<void>
```

Responds with autocomplete choices.

**Example:**
```typescript
if (interaction.isAutocomplete()) {
  const input = interaction.getInput();
  
  const choices = [
    { name: 'Apple', value: 'apple' },
    { name: 'Banana', value: 'banana' },
    { name: 'Cherry', value: 'cherry' }
  ].filter(c => c.name.toLowerCase().includes(input.toLowerCase()));
  
  await interaction.respond(choices.slice(0, 25)); // Max 25 choices
}
```
