## Overview

The `Command` class is the base class for creating slash commands (chat input commands) in Seyfert. It extends `BaseCommand` and provides the structure for handling application commands of type `ApplicationCommandType.ChatInput`.

## Class Signature

```typescript
export class Command extends BaseCommand {
  type = ApplicationCommandType.ChatInput;
  groups?: Parameters<typeof Groups>[0];
  groupsAliases?: Record<string, string>;
}
```

## Example

```typescript
import { Command, Declare, Options, type CommandContext } from 'seyfert';
import { ApplicationIntegrationType, InteractionContextType } from 'seyfert/lib/types';

const options = {
  user: {
    type: ApplicationCommandOptionType.User,
    description: 'The user to greet',
    required: true,
  },
};

@Declare({
  name: 'hello',
  description: 'Greet a user',
  integrationTypes: [ApplicationIntegrationType.GuildInstall],
  contexts: [InteractionContextType.Guild, InteractionContextType.BotDM],
})
@Options(options)
export default class HelloCommand extends Command {
  async run(ctx: CommandContext<typeof options>) {
    const user = ctx.options.user;
    
    await ctx.write({
      content: `Hello, ${user.username}!`,
    });
  }
  
  onRunError(ctx: CommandContext, error: unknown) {
    ctx.client.logger.error('Error in hello command:', error);
    ctx.editOrReply({
      content: 'An error occurred while executing the command.',
    });
  }
}
```

## SubCommand Class

For creating subcommands, use the `SubCommand` class:

```typescript
export abstract class SubCommand extends BaseCommand {
  type = ApplicationCommandOptionType.Subcommand;
  group?: string;
  declare options?: CommandOptionWithType[];
  
  abstract run(context: CommandContext): any;
}
```

<ParamField path="group" type="string">
  The name of the subcommand group this subcommand belongs to (optional).
</ParamField>

Subcommands must implement the `run` method and can access the same lifecycle hooks as the parent `Command` class.
