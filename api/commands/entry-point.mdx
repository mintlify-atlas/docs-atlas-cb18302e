## Overview

`EntryPointCommand` is the base class for creating primary entry point commands in Discord. These are commands that serve as the main entry point for activities and other app interactions.

## Class Signature

```typescript
export abstract class EntryPointCommand {
  middlewares: (keyof RegisteredMiddlewares)[] = [];
  name!: string;
  type = ApplicationCommandType.PrimaryEntryPoint;
  nsfw?: boolean;
  integrationTypes: ApplicationIntegrationType[] = [];
  contexts: InteractionContextType[] = [];
  description!: string;
  botPermissions?: bigint;
  dm?: boolean;
  handler!: EntryPointCommandHandlerType;
  name_localizations?: Partial<Record<LocaleString, string>>;
  description_localizations?: Partial<Record<LocaleString, string>>;
  props: ExtraProps = {};
  
  abstract run?(context: EntryPointContext): any;
}
```

### Type Guards

#### isEntryPoint

```typescript
isEntryPoint(): this is EntryPointContext<M>
```

Always returns `true` for EntryPointContext. Used for type narrowing.

#### inGuild

```typescript
inGuild(): this is GuildEntryPointContext<M>
```

Returns `true` if the command was invoked in a guild. Narrows type to `GuildEntryPointContext` which guarantees `guildId` and `member` are defined.

## GuildEntryPointContext

When `inGuild()` returns true, the context is narrowed to `GuildEntryPointContext`:

```typescript
export interface GuildEntryPointContext<M extends keyof RegisteredMiddlewares = never>
  extends Omit<MakeRequired<EntryPointContext<M>, 'guildId' | 'member'>, 'guild' | 'me'> {
  guild(mode?: 'rest' | 'flow'): Promise<GuildStructure<'cached' | 'api'>>;
  guild(mode: 'cache'): ReturnCache<GuildStructure<'cached'> | undefined>;
  
  me(mode?: 'rest' | 'flow'): Promise<GuildMemberStructure>;
  me(mode: 'cache'): ReturnCache<GuildMemberStructure | undefined>;
}
```

## Example

```typescript
import { EntryPointCommand, Declare, type EntryPointContext } from 'seyfert';
import { 
  ApplicationIntegrationType, 
  InteractionContextType,
  EntryPointCommandHandlerType,
} from 'seyfert/lib/types';

@Declare({
  name: 'My Activity',
  description: 'Launch my Discord activity',
  handler: EntryPointCommandHandlerType.AppHandler,
  integrationTypes: [ApplicationIntegrationType.GuildInstall],
  contexts: [InteractionContextType.Guild],
})
export default class MyActivityCommand extends EntryPointCommand {
  async run(ctx: EntryPointContext) {
    // Initialize activity
    await ctx.write({
      content: 'Starting activity...',
    });
    
    // Get guild information if needed
    if (ctx.inGuild()) {
      const guild = await ctx.guild();
      console.log(`Activity started in ${guild.name}`);
    }
    
    // Additional setup...
  }
  
  onRunError(ctx: EntryPointContext, error: unknown) {
    console.error('Failed to start activity:', error);
    ctx.editOrReply({
      content: 'Failed to start the activity. Please try again later.',
    });
  }
}
```

## Use Cases

Entry point commands are primarily used for:

- **Discord Activities**: Launch embedded activities (games, experiences)
- **App Entry Points**: Define how users can launch your app's main functionality
- **Interactive Experiences**: Create immersive, multi-user experiences in Discord
